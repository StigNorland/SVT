# `half_width = 8` Resolution Check

This note compares the first two constrained fine-grid runs at `half_width = 8`:

- `n = 48`
- `n = 64`

Artifacts:

- `papers/SSV-I/data/trefoil-state-n48-hw8-400steps-2026-05-17.json`
- `papers/SSV-I/data/trefoil-hw8-projected-residual-2026-05-17.json`
- `papers/SSV-I/data/q-p-static-potential-hw8-n48-2026-05-17.json`
- `papers/SSV-I/data/trefoil-state-n64-hw8-400steps-2026-05-17.json`
- `papers/SSV-I/data/trefoil-hw8-n64-projected-residual-2026-05-17.json`
- `papers/SSV-I/data/q-p-static-potential-hw8-n64-2026-05-17.json`

## Why This Matters

The `half_width = 8` box produced a much larger integrated proton source than the earlier `half_width = 6, 7` fine-grid states.

The key question was whether that jump was:

- a real large-box effect, or
- a coarse-grid artifact of using `n = 48`.

## Main Comparison

### Plateaued / near-plateau source values

For `n = 48`, the constrained continuation gives:

- `delta V_p ≈ 248.53`
- `Q_p^eff ≈ 246.03`

For `n = 64`, the constrained continuation gives:

- `delta V_p ≈ 249.63`
- `Q_p^eff ≈ 246.77`

So the source barely moves under the resolution increase from `48 -> 64`.

Approximate coarse-fine drift at fixed `half_width = 8`:

- `delta V_p`: about `0.4%`
- `Q_p^eff`: about `0.3%`

That is tiny compared with the earlier jump from `half_width = 7` to `8`.

### Other diagnostics

The shell density does shift:

- `n = 48`: about `0.9140`
- `n = 64`: about `0.9175`

So the outer recovery profile still changes somewhat with resolution.

But the integrated source does **not** move much, which is the important point for the gravity branch.

### Plateau behavior

`n = 48, half_width = 8` reaches plateau under the projected-residual criterion.

`n = 64, half_width = 8` also reaches plateau under the same criterion, with:

- projected residual dropping from about `0.0416` to `0.0280`
- two consecutive plateau hits by `700` total steps

## Interpretation

This resolves the main ambiguity.

The large `half_width = 8` source is **not** being generated by a low-resolution artifact at `n = 48`.

Increasing the resolution to `n = 64` leaves the integrated source essentially in the same regime.

So the current read is:

- the jump from `half_width = 7` to `8` is a real domain effect in the present model branch
- `half_width = 8` is numerically stable enough to treat as a serious reference point
- the real box-size question is now about the physics/domain trend itself, not about whether `n = 48` faked the result

## Consequence

The next useful comparison is no longer `n = 48` versus `n = 64` at `half_width = 8`.
That part is already informative.

The next useful comparison is the constrained fine-grid box-size trend across:

- `half_width = 6`
- `half_width = 7`
- `half_width = 8`

using the plateaued or source-flat states and comparing:

- `delta V_p`
- `Q_p^eff`
- shell density
- projected residual

That will tell us whether `8` is the start of a new asymptotic regime or whether the integrated source is still growing materially with box size.
